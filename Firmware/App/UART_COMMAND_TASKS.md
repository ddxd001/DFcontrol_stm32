# UART 指令与任务对应文档（详细动作版）

本文档与当前 `Firmware/App/app_tasks.c` 对齐，用于现场调试时快速确认每条指令的动作顺序与参数。

## 1. 协议与运行约束

- 帧格式：`AA BB XX`
- 解析任务：`App_Task_10ms()`
- 串口：`USART1`（`DebugUart`）
- 上电保护：前约 `1200ms` 触发请求会被清空
- 调度节拍：
  - 路口检测任务：`10ms`（100Hz）
  - 主状态机任务：`100ms`（等待时间以该节拍计数）

## 2. 指令总表

| 指令 | 功能 |
|---|---|
| `AA BB 01` | 蜂鸣器触发一次 |

| `AA BB 05` | 触发 Q3-1 |

## 3. 全局参数（当前代码）

- 常用距离：
  - `0.08m`（前置短移）
  - `1.0m`（标准段）
  - `0.92m`（Q1/Q4末段）
  - `0.9m`（Q2返程末段）
  - `0.55m` / `0.45m`（Q4中段）
  - `0.2m`（检测前段）
  - `0.4m`（检测段与检测后段，已更新）
  - `0.745m` / `0.47m` / `0.94m`（TEST3-x）
- 角度：
  - 左/右 `90°`
  - 调头 `180°`
  - `153.5°` / `116.5°` / `135°`（TEST3-x）
- 速度：
  - 自适应移动：`move_speed_mps100 = 3000`
  - 匀速“速度10”档：`move_speed_q3_case2_uniform_mps100 = 1000`
  - 检测段专用档：`move_speed_detect_mps100 = 500`（速度5）
- 常用等待：
  - 默认稳定等待：`1.4s`（14 ticks）
  - 检测段等待：`2.9s`（29 ticks）
  - 部分转向稳定等待：`0.5s`（5 ticks）

## 4. 各任务详细动作

### Q1（`AA BB 02`）

1. 蜂鸣一次后启动。
2. 前进 `0.08m`。
3. 循环 4 轮：左转 `90°` -> 直走。
4. 前 3 轮直走 `1.0m`，第 4 轮直走 `0.92m`。
5. 最后停车并蜂鸣。

### Q2-1（`AA BB 03`）

1. 蜂鸣一次后启动。
2. 前进 `0.08m`。
3. 循环 3 轮：左转 `90°` -> 直走 `1.0m`。
4. 调头 `180°`。
5. 返程 3 段：`1.0m` + `1.0m` + `0.9m`。
6. 停车并蜂鸣。

### Q2-2（`AA BB 04`）

与 Q2-1 相同，仅首段转向为右转 `90°`。

### Q3-1（`AA BB 05`）

1. 总体结构与 Q1 相同，按轮次运行。
2. 第 2 次直走（`s_round == 1`）替换为“测距流程”（与 TEST1 同构）：
   - 读取灰度偏移并做小角度修正（若偏差足够大才修正）。
   - 修正后稳定等待 `0.5s`。
   - 匀速前进 `0.2m`（速度10）。
   - 反向转回修正角，稳定等待 `0.5s`。
   - 蜂鸣一次。
   - 匀速前进 `0.4m`（检测段，速度5，开启路口检测与测距）。
   - 稳定等待 `2.9s`（检测窗口）。
   - 蜂鸣两次。
   - 自适应前进 `0.4m`，结束该替换段。
3. 替换段结束后回到 Q3-1 主循环继续，最终停车并蜂鸣。

### Q3-2（`AA BB 06`）

1. 蜂鸣一次后启动。
2. 匀速前进 `1.0m`（速度10）。
3. 稳定等待后停车并蜂鸣。

### Q4（`AA BB 07`）

1. 蜂鸣一次后启动。
2. 前进 `0.08m`。
3. 左转 `90°`，稳定等待。
4. 前进 `0.55m`。
5. 等待 `4.0s`。
6. 前进 `0.45m`。
7. 左转 `90°`（同时下发步进电机右转90°）。
8. 前进 `0.92m`。
9. 停车并蜂鸣。

### TEST1（`AA BB D0`）

1. 蜂鸣一次后启动。
2. 检测前修正段：
   - 灰度偏移修正（可选）。
   - 修正稳定等待 `0.5s`。
   - 匀速前进 `0.2m`（速度10）。
   - 反向转回修正角并稳定等待 `0.5s`。
3. 测距主段：
   - 蜂鸣一次。
   - 匀速前进 `0.4m`（速度5，开启路口检测与测距）。
   - 稳定等待 `2.9s`（检测窗口）。
4. 收尾段：
   - 蜂鸣两次。
   - 自适应前进 `0.4m`。
   - 停车并蜂鸣。

### TEST2（`AA BB D1`）

1. 蜂鸣一次后启动。
2. 匀速前进 `1.0m`（速度10）。
3. 稳定等待后停车并蜂鸣。

### TEST3-1（`AA BB D2`）

## 5. 路口检测与测距（TEST1 + Q3-1替换段）

- 触发窗口：仅在检测段（匀速 `0.4m` + 等待窗口）内有效。
- 判定方法：`GwGray_ReadAndDetectCross()`，阈值 `APP_GRAY_CROSS_MIN_ACTIVE_BITS = 3`。
- 计数方式：锁存上升沿计数（`latched`），每次新路口蜂鸣一次。
- 路口上限：`APP_MEAS_EXPECT_CROSS_COUNT = 3`，达到后停止后续检测。
- 串口输出策略：检测期间先缓存，检测段结束后统一发送。
  - `CROSS:<n>`
  - `DIST:xx.xcm`
- 当前测距公式（相邻路口）：
  - `距离(cm,保留1位) = dt_ms * 0.15 * 0.798 * 100 * 10 / 1000`
  - 整数实现：`dist_cm_x10 = (dt_ms * 1197 + 5000) / 10000`
- 距离钳位：`1.0cm ~ 5.0cm`（超出范围按边界值输出）。

## 6. 步进电机（`08/09/0A/0B`）

- 驱动：`StepperMotorDrv`
- 串口：`USART2`（`PA2 TX / PD6 RX`）
- 指令行为：
  - `08`：使能
  - `09`：失能
  - `0A`：左转90°（相对位移，`pulse = -800`）
  - `0B`：右转90°（相对位移，`pulse = +800`）
- 速度参数：`100`

## 7. 串口输出

- 路口事件：`CROSS:<n>`
- 路口间距：`DIST:xx.xcm`（仅 TEST1 与 Q3-1 替换段）
